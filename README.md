# MLB 선발투수 컨디션 조기 예측

## 프로젝트 개요

본 프로젝트는 **MLB 선발투수의 경기 초반 투구 데이터를 활용하여 그날 남은 경기의 컨디션(헛스윙 유도력)을 예측**하는 것을 목표로 합니다.
경기 초반 15구(X구간)에서 이미 나타나는 신호를 바탕으로, 실점이 나오기 전에 투수의 당일 컨디션을 조기 판단하고, 컨디션에 영향을 주는 주요 투구 인자를 찾아 조기 교체·불펜 준비 등 벤치 의사결정에 활용하고자 합니다.

핵심 철학은 이 투수가 **"누구인가"(통산 성적·이름값)가 아니라 "오늘 어떤 상태인가"(컨디션)** 를 예측하는 것입니다.

## 프로젝트 목표

- 경기 초반 15구만으로 **이후 타석의 whiff%(헛스윙률)를 예측**하는 회귀 모델 구축
- 컨디션에 영향을 미치는 **주요 투구 인자 파악**(SHAP 기반 해석)
- **"오늘 컨디션이 좋은 날인가"를 이진 분류**로 실용화 (조기 교체 판단 지원)
- 세운 가설을 **통계적으로 검증**하고, 효과 없는 접근은 정직하게 기각

## 수행 내용

- Baseball Savant Statcast Data **EDA(탐색적 데이터 분석)**
- **타겟 지표 선정** — 예측 가능성 기반으로 예측 대상(Y) 재정의
- **X구간 실험** — 경기 초반 어디까지 봐야 예측이 되는지 탐색
- **Feature Engineering** — 무브먼트·회전축·부하(ACWR) 등 도메인 feature 설계
- **결측·이상치 처리 전략 비교**
- **Optuna 하이퍼파라미터 최적화**
- **paired t-test 기반 가설 검증** (모든 feature 채택 결정에 적용)
- **영상 생체역학 파이프라인** 구축 및 융합 실험 (MediaPipe 관절 추출)
- **SHAP 기반 모델 해석** 및 투수 유형별 오차 분석
- **컨디션 이진 분류** 모델 구축

### 예측 대상 (Y)

```
whiff% = swinging_strike 수 / 전체 스윙 수
전체 스윙 = swinging_strike + swinging_strike_blocked + foul + foul_tip + hit_into_play
- 스윙 20개 미만 경기 제외 (분모 안정성)
```

## 주요 결과

### 1. 예측 대상(Y) 재정의 — 예측 가능한 문제 찾기

모델을 개선하기 전에 "이게 애초에 예측 가능한 문제인가"를 상관 분석으로 먼저 검증했습니다. 투수의 대표 지표인 **ERA부터 시작**해 4개 지표를 비교했습니다.

| 타겟 (Y) | 최고 상관 (구속과) | Val R² | 판정 |
|---|---|---|---|
| ERA (평균자책, 대표지표) | 0.064 | 0.0006 | 수비·운 노이즈 과다 (DIPS 이론) |
| FIP (수비무관 자책) | 0.101 | 0.0093 | 여전히 결과 기반 |
| xwOBA (기대 가중출루율) | 0.105 | 0.0211 | 타구 질로 운 일부 제거 |
| **whiff% (헛스윙률)** | **0.213** | **0.0934** | **투수가 직접 통제 → 채택** |

ERA→whiff%로 갈수록 상관이 **약 3배(0.064→0.213)**, Val R²가 **150배 이상(0.0006→0.093)** 상승했습니다. 결과(안타·실점)에 의존하는 지표일수록 운·수비가 섞여 예측 불가였고, 투수가 직접 통제하는 헛스윙률만 예측 가능했습니다.

| Feature 수 | **79** |

> R²가 낮아 보이지만, 선행 연구에서도 구속으로 ERA를 예측하면 R² ≈ 0.05~0.08, 최고 지표 SIERA조차 다음 시즌 ERA의 20%만 설명합니다. 경기 단위 + 초반 15구라는 훨씬 어려운 조건에서 Val R² 0.09는 **문제의 천장에 근접한 값**입니다. ([98_참고논문/08](98_참고논문/08_추가수집_타겟평가지표.md) 참조)

### 3. 무브먼트 feature — 최대 성능 개선

기존 모델은 구속·회전수·릴리스 위치는 담았으나, **"공이 실제로 얼마나 휘었나(무브먼트, pfx)"** 라는 헛스윙의 직접적 원인이 빠져 있었습니다. 무브먼트·회전축·제구 feature를 단계적으로 편입한 결과:

| 모델 | feature | Test R² | 개선 |
|---|---|---|---|
| 정형 baseline + ACWR (v2) | 62 | 0.0274 | — |
| + 무브먼트·회전축·제구 (v3) | 75 | 0.0390 | +42% (p=3e-33) |
| **+ 무브먼트 gap (v4)** | **79** | **0.0399** | Val +0.0012 (p=0.001) |

100-seed paired t-test로 재현성·회귀·분류를 전부 통과한, 프로젝트 전체 최대 개선입니다.

**무브먼트 gap — 쪼개기 vs 차이**: v3의 무브먼트는 구종 3그룹 *평균*이라, 그룹 안에서 정보가 뭉개집니다. 이를 복원하는 방법으로 "구종을 더 잘게 쪼개기"(직구 3종·8구종)와 "구종 간 차이(gap, 직구−변화구 무브먼트)"를 비교했는데, **쪼개기는 표본 손실로 재현성에 실패하고 gap만 통과**했습니다. gap은 안정적인 두 그룹 평균의 차이라 표본 손실 없이 "구종 간 궤적 대비"라는 새 정보를 더합니다 — 헛스윙은 "직구인 줄 알고 스윙했는데 변화구가 와서" 나오므로, 궤적 차이가 클수록 유인력이 높아집니다. ([4_output/movement_gap_results.md](4_output/movement_gap_results.md) 참조)

### 4. 컨디션 이진 분류 — 실용화

정확한 whiff% 값 예측은 어렵지만, "오늘 컨디션이 좋은 날인가"로 문제를 바꾸면 실용적인 성능이 나옵니다 (whiff% 상위 33% = 좋은 날 / 하위 33% = 나쁜 날).

| 모델 | Val AUC | Test AUC | '좋은 날' Recall |
|---|---|---|---|
| XGBClassifier (정형) | 0.651 | 0.597 | 74% |

## 영상 생체역학 파이프라인

Statcast는 "공의 결과"(구속·회전수)는 정밀하게 담지만 **투수의 몸이 어떻게 움직였는지**는 담지 않습니다. "폼이 무너졌다"는 감각을 데이터로 잡아낼 수 있는지 검증하기 위해, 투구 영상에서 관절 좌표를 추출하는 파이프라인을 설계했습니다.

```
9초 클립 다운로드
  → 컷 분할 (PySceneDetect) → 투구 장면 필터링 (YOLOv8 사람 검출 + 그라운드 비율)
  → 투수 선택 (전신 키포인트 + 화면상 y좌표 최대)
  → 릴리스 프레임 특정 (팔 뻗음 × 속도 최대)
  → MediaPipe Pose 14개 관절 좌표 추출
  → 좌투 미러링 → 어깨너비 정규화 → 이상치 윈저라이징
  → 각도 9종 계산 (stride, arm_slot, shoulder_tilt, trunk_angle, separation ...)
  → 경기 단위 집계 (mean/std/full9 × 초반 3/5/10/15구)
```

## 가설 검증 — 정직한 기각

성능을 부풀리는 대신, 세운 가설을 **30~100 seed paired t-test**로 검증하고 효과 없는 것은 기각했습니다.

| 가설 | 검증 결과 | 판정 |
|---|---|---|
| delta(평소 대비 편차) feature | 차이 ≈ 0, 절반은 음수 | 기각 |
| **영상 생체역학 융합** | ΔR² -0.0092, **p ≈ 9.6×10⁻⁷ (유의 악화)** | 기각 |
| 결측 처리 5종 / 이상치 처리 4종 | baseline이 최선 | 기각 |
| 시퀀스 모델(1D-CNN) / changepoint | 유의한 개선 없음 | 기각 |
| 구종 세분화(직구 3종·8구종) | 재현성 실패 | 기각 |

> 같은 "무브먼트를 더 쓰자"는 아이디어여도, **구종을 쪼개는 방식은 표본 손실로 재현성에 실패**했지만 **구종 간 차이(gap)는 재현성을 통과**해 v4에 채택됐습니다(3장 참조). 접근 방식에 따라 재현성이 갈린 대표 사례입니다.

<img src="4_output/figures/shap_bio_comparison.png" width="680">

> 좌: SHAP Top 30 (파랑=정형, 주황=영상) — 영상 feature는 11위부터 등장. 우: feature 그룹별 평균 |SHAP| — 정형(0.00070)이 영상(0.00032)의 2배 이상.

**영상 융합 기각 과정의 교훈 — SHAP 중요도 ≠ 예측 기여**: 영상 피처가 SHAP 중위권에 올라도 실제 융합 시 성능이 떨어졌습니다. 1차(절반 데이터)에서 "무의(p>0.05)"였던 결론이 전체 데이터(3.3배)로 재검증하자 "유의하게 악화"로 명확해졌습니다. 표본을 늘려 결론의 신뢰도를 높인 과정 자체가 검증 능력의 근거입니다.

## 한계점

- **경기 단위 예측의 구조적 천장**: 야구 경기 단위 성과는 상대 타자·그날 스트라이크존 등 투수가 통제할 수 없는 변동이 커, 초반 15구만으로는 Val R² 0.09 수준이 한계입니다(선행 연구의 구속→ERA R² 0.05~0.08과 유사한 범위). 값 자체의 정밀 예측보다 "좋은 날/나쁜 날" 이진 분류(AUC 0.65)가 더 실용적인 이유입니다.
- **핀포인트형 투수 예측의 한계**: 모델은 구속·무브먼트 신호가 뚜렷한 파워형에는 잘 맞지만, 위력이 아닌 제구로 승부하는 핀포인트형에는 예측력이 떨어집니다(Test R² -0.024). 유형별 분리 모델링이 필요한 지점입니다.
- **영상 생체역학의 한계**: 릴리스 순간의 "정지 1프레임 자세 각도"는 whiff%와 인과가 약해 융합 실험이 통계적으로 기각됐습니다. 자세의 *변화·타이밍*을 담으려면 릴리스 전후 시퀀스(동작 궤적) 접근이 필요하나, 이는 별도 연구 범위입니다.
